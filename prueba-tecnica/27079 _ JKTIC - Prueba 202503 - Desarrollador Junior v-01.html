Prueba técnica - Desarrollo de un Login

 

Prueba básica en para evaluar las habilidades en la creación de un login. La prueba cubre:

  • Manipulación del DOM.
  • Manejo de eventos.
  • Uso de localStorage para persistencia simple. (O la que proponga el aspirante si usa algun Framework)
  • Validación básica de campos.

Objetivo:

Construir un formulario de inicio de sesión . Se debe validar el usuario y la contraseña, y simular un inicio de sesión exitoso almacenando datos en localStorage.

Instrucciones:

  1. Crear un formulario de login con los siguientes elementos:
    • Un campo para ingresar el usuario.
    • Un campo para la contraseña.
    • Un botón de Iniciar Sesión.
    • Incluir una imagen en el diseño
  2. Validar los datos ingresados:
    • Usuario: debe ser un correo electrónico válido.
    • Contraseña: mínimo 6 caracteres.
  3. Simular la autenticación:
    • Si el usuario y la contraseña son correctos, almacenar el usuario en localStorage y mostrar un mensaje de bienvenida.
    • Si los datos son incorrectos, mostrar un mensaje de error.
  4. Persistencia de sesión:
    • Si el usuario ya ha iniciado sesión previamente, mostrar directamente el mensaje de bienvenida.

Criterios de Evaluación

  • Uso correcto de JavaScript para manipular el DOM.
  • Aplicación de validaciones adecuadas.
  • Implementación de localStorage para la persistencia de sesión.
  • Código limpio, estructurado y con buenas prácticas.

 

Extras (Para aspirantes avanzados)

Si deseas evaluar más a fondo, puedes pedir que agreguen:

  • Un botón de Cerrar Sesión que borre el usuario de localStorage.
  • Un efecto de loading en el botón mientras se procesa el login.
  • Uso de fetch() para autenticar con una API (simulada con setTimeout).
  • Un diseño moderno y Responsive
  • Controlar el proyecto con Git y subirlo a un repositorio publico en github

 

Puede ayudarse de IAs para la ejecución de la prueba pero debe contar con la capacidad para explicar el código en el momento de la entrevista
